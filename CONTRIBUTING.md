# CarWheels Contributing Guide

Thank you for your interest in contributing to CarWheels! We welcome contributions from everyone.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourname/CarWheels.git`
3. Create feature branch: `git checkout -b feature/your-feature`
4. Make your changes
5. Push: `git push origin feature/your-feature`
6. Open Pull Request

## Development Setup

```bash
npm run install-all
npm run dev
```

## Code Standards

- Use TypeScript for type safety
- Follow existing code style
- Add comments for complex logic
- Write unit tests for new features
- Update documentation

## Commit Messages

```
feat: Add new feature
fix: Fix bug
docs: Update documentation
refactor: Refactor code
test: Add tests
chore: Update dependencies
```

## Pull Request Process

1. Update README.md with changes
2. Ensure tests pass
3. Add screenshots for UI changes
4. Describe your changes clearly
5. Link relevant issues

## Reporting Bugs

Include:
- Description of the bug
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots/logs

## Feature Requests

Include:
- Clear description
- Use case and benefit
- Potential implementation approach
- Any relevant examples
